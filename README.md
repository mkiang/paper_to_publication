# Paper to Publication

This is a companion repository for my [IAPHS How-To Tuesday](https://iaphs.org/tools-for-success/online-events/) talk, "How to go from paper to publication". 
It includes additional resources (below) as well as the slides, data, and code used for the talk. Slides can be found in [`./talk`](https://github.com/mkiang/paper_to_publication/tree/main/talk), code in [`./code`](https://github.com/mkiang/paper_to_publication/tree/main/code), and data in [`./data`](https://github.com/mkiang/paper_to_publication/tree/main/data).

## Disclaimer 

All advice here reflects experience as a public health researcher, section editor at a digital health journal, and associate editor at a clinical/subspecialty journal. Norms vary across fields — take what's useful, leave what isn't. Reasonable people in the same field will disagree on some of this.

### Cover letter template (4 paragraphs, 1 page)

The structure I use for nearly every submission. Tailor to the target journal each time.

**Paragraph 1 — Hook and importance.** State the problem in two sentences. Why does this matter now? Avoid restating the abstract.

**Paragraph 2 — Headline findings.** Two or three high-level results in your own words. Methods at one-line resolution ("we used stochastic dynamical models"). No parameter details — the manuscript covers that.

**Paragraph 3 — Show the fit.** Cite recent papers from *this* journal that your work builds on, extends, or speaks to. If you can't find any, it may be the wrong journal.

**Paragraph 4 — Logistics.** Not under consideration elsewhere. Preprint status. Conflicts of interest. Any prior editor correspondence.

Constraints: one page, no jargon, written for an editor who knows the field but not your subarea.

### Writing a response letter

- Noble WS. *Ten simple rules for writing a response to reviewers.* PLOS Computational Biology. 2017;13(10):e1005730. doi:[10.1371/journal.pcbi.1005730](https://doi.org/10.1371/journal.pcbi.1005730). PMC5638205. **Read this paper at least once.**
- Hiemstra PS. *How to write a response to the reviewers of your manuscript.* Breathe. 2018;14(4):319-321. doi:[10.1183/20734735.025818](https://doi.org/10.1183/20734735.025818).
- Rusk N. *How to write a rebuttal letter.* Research Communities by Springer Nature [blog post]. September 27, 2013. URL: <https://communities.springernature.com/posts/how-to-write-a-rebuttal-letter>.

### Response letter template

Adapted from Noble (2017): 

1. **Make the response self-contained.** Copy-paste each critique verbatim, then respond directly underneath. Editor should be able to assess whether you addressed the comment without opening the manuscript.
2. **Respond to every point.** Number reviewer critiques and split out double-barrelled comments. Don't skip points you disagree with — name the disagreement and move on.
3. **Do what the reviewer asks.** Even if you think the request won't change anything. Show that it doesn't matter rather than asserting it.
4. **Get it done now.** Speed preserves reviewer memory of the first round and reduces the chance they ask for new things in round two. In my lab, an R&R is an all-hands sprint.
